A6 Activity 개요

A6고객 측 준비 현황·보안·리스크 점검

착수 후 반복적으로 발생하는 지연 요인을 Sprint 0에서 조기에 식별하, 담당과 기한을 지정하여 본수행 착수 판단에 연결합니다.

핵심 질문
본수행을 지연시킬 고객·보안·데이터 리스크는 무엇인가?

고객 담당자 데이터 일정 보안 기준 이슈·리스크 Open Item
A6 결정 항목
본수행을 막을 고객·데이터·보안 리스크를 사전 식별합니다.

고객 참여, 데이터 제공, 보안 기준, 주요 이슈·리스크, 개발 시작 차단요소를 확인하여 Sprint 1 이전에 지연 요소을 파악합니다. 이 Activity의 결과는 점검표 생성이 아니라, Sprint 1 시작 판단을 위한 리스크 기준을 정하는 것입니다.

구분결정해야 할 내용
고객 준비상태고객 담당자, 승인자, 참여 가능 시간
데이터 준비상태데이터 제공 일정, 마스킹 기준, 원천 접근 가능성
보안 준비상태보안교육, 개인정보, 자료 반출입, 망접속 기준
이슈·리스크 등급영향도, 발생가능성, 대응전략
Sprint 1 시작 차단 여부조건부 진행 / 시작 전 확인 필요 후보 여부

완료 기준
고객·데이터·보안 준비상태가 확인되고, 모든 이슈·리스크 항목에 담당과 기한이 있어야 합니다. 미확정 항목은 이행 확인 체크리스트에서 미흡 항목으로 표시하고 Sprint 1 실행준비 및 미정 항목 이관 점검에서 재확인합니다.

1제목

이행 확인 체크리스트

Task별 Play Guide를 진행한 뒤, 그 결과가 실제 프로젝트 운영에 반영되었는지 아래 기준과 근거로 확인합니다.

확인 기준미흡 시 조치
고객 담당자, 승인자, 업무 담당자의 참여 가능 일정이 확인되어 있는가수행PM이 고객 PM에게 참여자 일정 재확인을 요청하고, 3일 이내 확정한다
데이터 제공 일정, 제공 방식, 마스킹 기준이 확인되어 있는가수행PM이 고객 데이터 담당자에게 제공 일정과 책임자를 확인 요청하고, 데이터 제공 계획에 반영한다
보안·개인정보·자료 반출입 기준이 확인되어 있는가수행PM이 고객 보안 담당자에게 기준 확인을 요청하고, 확인 결과를 후속 점검 항목으로 지정한다
주요 이슈와 리스크가 영향도 기준으로 구분되어 있는가PL이 이슈·리스크 목록의 영향도를 재분류하고, 수행PM이 대응 우선순위를 확정한다
Sprint 1 시작을 막을 수 있는 차단 리스크가 별도로 식별되어 있는가수행PM이 차단 리스크를 별도 목록으로 정리해 Sprint 1 실행준비 및 미정 항목 이관 점검에서 시작 전 확인 필요 후보로 반영한다
고객 가치(Customer Value) 확인 관점

고객 가치(Customer Value) 확인을 방해하는 리스크는 무엇인가

고객 가치 확인이 실패하는 주요 원인은 개발 역량 부족이 아니라 고객 참여 부족, 데이터 미확보, 보안 제약, 현업 일정 부족, 의사결정 지연입니다. 이 Activity의 전체 이슈·리스크 관리와 중복하지 않고, 고객 가치 확인을 직접 막는 항목만 별도로 확인합니다.

구분정리 정보
참여 리스크현업 인터뷰/관찰 참여 불가 가능성
데이터 리스크데이터 제공 지연, 비식별화(마스킹), 반출 제한
보안 리스크화면 확인, 자료 반출, 외부망 접근 제약
의사결정 리스크의사결정 책임자(Sponsor), 승인자, 우선순위 결정자 부재
대체 방안샘플 데이터, 가상 자료(Mock Data), 제한된 관찰, 비식별 자료
진행 방식: 일반 리스크 목록을 다시 만들지 않습니다. 앞서 진행한 이슈·리스크 점검 중에서 고객 가치 확인을 직접 막는 항목만 별도로 표시합니다.
