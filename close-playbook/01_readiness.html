Activity 1·2·3 — 종료 준비

종료 전
무엇을, 어떻게 확인하는가

Close 준비 착수(A1), Scope·계약·변경 이행 점검(A2), 요구사항·Backlog·WBS 완료 점검(A3)의 체크 항목·양식·판단 기준을 제공합니다.

핵심 원칙
요구사항 완료가 아니라
"요구사항 설명 가능성"이 기준입니다.
고객에게 이렇게 설명할 수 있어야 합니다:
"이 요구사항은 어떤 산출물로 구현되었고, 어떤 테스트로 검증되었으며, 고객이 언제 확인했습니다."

Backlog 종료 상태 분류

Done

Acceptance Criteria 충족, 고객 확인 완료.

Deferred

후속 단계 이관. 고객 합의 + Owner + Due Date 필수.

Cancelled

범위 제외. Scope Out 또는 Waive 근거 필수.

주의: 미완료 항목을 0으로 만들어야만 종료할 수 있는 것은 아닙니다. 단, 모든 미완료 항목은 반드시 완료 / Deferred / Cancelled / Waived 중 하나로 분류되어야 합니다. 분류 없는 미완료는 종료 보류 조건입니다.
