A2 Activity 개요

A2Project Offboarding

A2 Project Offboarding은 고객사 종료보고 이후 프로젝트팀이 고객 환경과 프로젝트 수행 체계에서 안전하게 철수하기 위한 Activity입니다.

이 단계는 단순 인력 철수가 아니라 운영/하자보수 이관, 권한 회수, 고객 자료 반납/폐기, 잔여사항 이관, 역할 종료를 포함합니다.

프로젝트팀이 고객사와 수행 환경에서 안전하게 철수할 준비가 되었는가?

1제목

이행 확인 체크리스트

A2가 실제로 이행되었는지 확인하는 기준

확인 기준미흡 시 조치
철수 일정과 대상자가 정리되었는가?PM이 인력별 마지막 근무일과 인수인계 대상을 조직도에 표기하고, 후속 문의 연락처(PM 또는 지정 담당자)를 팀 전체에 공지한다.
운영 이관 항목이 완료되었는가?PL이 운영 담당자와 운영자 매뉴얼(Runbook)·모니터링 항목을 하나씩 시연하며 인수 서명을 받고, 미인수 항목은 인수 완료일을 재지정한다.
하자보수 전환 기준이 명확한가?PM이 계약서상 하자보수 조건(기간·무상범위)을 근거로 접수 채널과 담당자를 확정하고 고객에게 서면 안내한다.
권한 회수 대상이 누락 없이 정리되었는가?보안 담당자가 Sprint 0 A5에서 확보한 권한 목록(협업도구·고객 시스템·VPN·VDI·저장소·출입증)을 대조해 미회수 항목을 특정하고, 회수 후 스크린샷·로그를 증적으로 보관한다.
고객 자료와 보안자료가 정리되었는가?보안 담당자가 로컬·공유 저장소의 고객 자료를 전수 확인해 반납 또는 삭제하고, 삭제 확인서에 서명받는다.
잔여사항 이관 대상이 명확한가?PM이 잔여사항별로 이관 대상 조직(운영/하자보수/고객)과 담당자·기한을 지정하고 해당 조직의 인수 확인을 받는다.
