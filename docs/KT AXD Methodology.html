KT Internal Methodology · AI-Driven Delivery Lifecycle

AI-Driven Delivery, the AXD way.

Agent AI를 중심에 둔 4단계 딜리버리 라이프사이클. Discovery에서 Continuous Improvement까지, AI 모델 정책·평가·운영 데이터를 기반으로 지속 고도화되는 KT AXD 방법론입니다.

01
Phase / Stage
Discovery · 현행 분석 & 요구사항
현행 시스템을 분석하고 유저스토리를 매핑하며 Agent AI의 활용 시나리오와 모델 정책을 명세하는 단계. 이 단계에서 평가 전략과 요구사항 추적 체계를 확립하여 이후 Sprint의 품질 기준을 세운다.
D-01
현행시스템 리뷰
As-Is System Review
기존 시스템의 아키텍처·데이터·인터페이스를 분석하여 Agent AI 도입 영향 범위를 파악한다.
Phase 01 →
D-02
유저스토리 매핑
User Story Mapping
사용자 여정을 기반으로 에픽·스토리를 구조화하여 Agent AI 서비스와 연결한다.
Phase 01 →
D-03
Agent AI 활용 시나리오 정의
Agent AI Use Case Scenario
업무 도메인별 Agent AI 적용 시나리오를 정의하고 우선순위를 결정한다.
Phase 01 →
D-04
Agent AI 서비스 정의
Agent AI Service Definition
제공할 Agent AI 서비스의 범위·기능·인터페이스를 명세한다.
Phase 01 →
D-05
Agent AI 모델 정책 명세
Model Policy Specification
모델 선택 기준, 버전 관리, 안전 정책, 할루시네이션 대응 가이드라인을 수립한다.
Phase 01 →
D-06
평가 전략 수립
Evaluation Strategy
Agent AI 품질 평가 지표(정확도·응답시간·안전성)와 평가 주기를 정의한다.
Phase 01 →
D-07
요구사항 추적
Requirements Traceability
요구사항–설계–구현–테스트 간 추적 매트릭스를 수립하여 누락·변경을 관리한다.
Phase 01 →
D-08
Discovery 단계 점검
Discovery Gate Review
Discovery 단계 완료 기준(DoD)을 점검하고 Phase 2 진입을 승인한다.
Phase 01 →
02
Phase / Stage
Design Sprint · Build & Experiment · Evaluate & Validate
Agent 아키텍처를 설계하고 DataOps·AgentOps·서비스 관리도구를 Sprint 단위로 구현·실험·검증하는 핵심 단계. PI 계획 기반으로 반복적 빌드와 평가를 수행하며 품질 게이트를 통과해 나간다.

B-01
Agent 아키텍처 설계
Agent Architecture Design
Agent 오케스트레이션·도구·메모리·프롬프트 체계를 설계한다.
Phase 02 →
B-02
평가기준 정의
Evaluation Criteria Definition
Sprint별 Agent 출력 품질 기준과 자동화 평가 파이프라인 설계.
Phase 02 →
B-03
PI 및 Sprint 계획 수립
PI & Sprint Planning
Program Increment 단위로 목표를 설정하고 Sprint 백로그를 계획한다.
Phase 02 →
B-04
단계별 품질 점검
Stage Quality Gate
각 Sprint 종료 시 DoD 충족 여부와 품질 지표를 검토한다.
Phase 02 →
B-05
Agent AI 평가
Agent AI Evaluation
정의된 평가 기준에 따라 Agent 응답 품질·안전성·성능을 측정한다.
Phase 02 →
B-06
통합·회귀 테스트
Integration & Regression Test
컴포넌트 간 통합 및 기존 기능 회귀 여부를 자동화 테스트로 검증한다.
Phase 02 →
B-07
E2E 시스템 테스트
End-to-End System Test
실제 사용자 시나리오 기반으로 전체 흐름을 검증한다.
Phase 02 →
B-08
보안 취약점 점검
Security Vulnerability Assessment
Prompt Injection·데이터 유출·접근 제어 등 AI 특화 보안 위협을 점검한다.
Phase 02 →
03
Phase / Stage
Release & Operate · 릴리즈 & 운영
사용자 교육부터 시스템 전환·오픈까지. 인수 검증을 완료하고 안정화 기간을 거쳐 운영팀으로 인수인계하며 서비스를 공식 오픈한다.
R-01
사용자 교육
User Training
최종 사용자 대상 Agent AI 서비스 사용 교육 및 매뉴얼을 제공한다.
Phase 03 →
R-02
인수 및 검증 테스트
User Acceptance Test (UAT)
고객이 직접 비즈니스 시나리오를 검증하며 최종 인수 승인을 수행한다.
Phase 03 →
R-03
릴리즈 준비·환경 계획
Release Readiness & Environment Plan
운영 환경 구성, 롤백 계획, 릴리즈 체크리스트를 완비한다.
Phase 03 →
R-04
시스템 전환·오픈
System Cutover & Go-Live
기존 시스템에서 새 Agent AI 서비스로 전환하고 공식 오픈을 선언한다.
Phase 03 →
R-05
안정화 및 인수인계
Stabilization & Handover
오픈 후 안정화 기간 동안 이슈를 신속 대응하고 운영팀에 지식·문서를 이관한다.
Phase 03 →
04
Phase / Stage
Continuous Improvement · AIDD Loop
단순 종료가 아닌, 운영 데이터를 기반으로 AI 모델·프롬프트·정책을 지속 개선하는 선순환 루프. AIDD(AI-Driven Development) 기반으로 산출물·품질·개발 자동화를 고도화하며 다음 Discovery 사이클로 연결된다.
C-01
Lessons Learned
회고 및 교훈 도출
프로젝트 전반의 성공·실패 요인을 분석하고 조직 학습 자산으로 전환한다.
Phase 04 →
C-02
AI 모델 성능 개선
AI Model Performance Improvement
운영 지표를 기반으로 모델 버전 업그레이드·파인튜닝 계획을 수립한다.
Phase 04 →
C-03
Prompt / 정책 개선
Prompt & Policy Optimization
실사용 피드백·에러 케이스를 분석하여 프롬프트와 운영 정책을 최적화한다.
Phase 04 →
C-04
운영 데이터 기반 재학습
Operational Data Re-training
축적된 운영 데이터를 활용해 모델을 재학습하거나 RAG 지식베이스를 갱신한다.
Phase 04 →
C-05
AIDD 기반 자동화 고도화
AIDD Automation Enhancement
AI-Driven Development 기반으로 산출물·품질·개발 자동화 수준을 지속 향상한다.
Phase 04 →
