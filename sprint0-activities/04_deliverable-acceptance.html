A4산출물·검토·승인 기준 정렬

전통적 사업수행계획서의 산출물 목록을 Hybrid 수행에 맞게 테일러링하고, 각 산출물이 언제 어떤 기준으로 승인되는지 명확히 합니다.

공통 운영 기준: Sprint 0 · 작업 목록 · 작업 시작·완료 기준 · 검수·수용 기준 AXD 수행계획서 Chapter 연결 본수행 착수점검 착수 판단 입력
Playbook Output
Task를 수행하면 템플릿이 생성되고, 템플릿 상태가 본수행 착수 판단으로 연결됩니다.

이 HTML Pack은 전통적 사업수행계획서 산출물을 대체하기보다, Sprint 0에서 Hybrid 수행에 필요한 입력값과 실행 산출물을 구조화하는 용도입니다.

Task

수행 Task

이 Activity를 수행할 때 실제로 처리해야 할 작업입니다.

1산출물 목록 테일러링

사업관리 산출물, 개발 산출물, Agile 운영 산출물을 필수/조건부/선택/제외로 분류합니다.

2작성·검토·승인자 지정

산출물별 Owner, Reviewer, Approver를 지정합니다.

3제출 시점 정의

단계, Sprint, Gate, 고객 검토 시점별 제출 일정을 정의합니다.

4검수·수용 기준 정의

산출물 승인 기준, 반려 기준, 보완 기한을 정합니다.

5형상관리 연결

문서 ID, 버전, 저장 위치, 변경이력 기준을 정합니다.

Play 방식

어떻게 수행하는가

  • 기존 AXD 사업관리/개발 산출물 목록을 출발점으로 삼습니다.
  • 프로젝트 유형에 맞지 않는 산출물은 “제외” 처리하고 사유를 남깁니다.
  • 산출물 내용은 Sprint 0에서 완성하지 않지만, 승인 기준은 Sprint 0에서 정합니다.
  • 고객 승인자가 없는 산출물은 본수행 착수점검 체크에서 조건부 또는 착수 보류 항목으로 관리합니다.
