Sprint 0 Overview

Sprint 0 Activity Playbook

Sprint 0 본문과 7개 하위 Activity별로 PM이 무엇을 질문하고, 결정하고, 기록해야 하는지 안내하는 실행 가이드입니다.

결정해야 할 것 수행 Task별 Play Guide 이행 확인 체크리스트
Read-only Playbook
Task를 수행한 결과가 결정사항으로 정리되고, 그 결정사항이 본수행 착수 판단으로 연결됩니다.

이 Playbook은 입력·저장 시스템이 아니라, PM이 Sprint 0를 진행할 때 확인해야 할 질문, 결정사항, 이행 확인 기준을 제공하는 참고 가이드입니다. 정리된 결과는 사업수행계획서, 본수행 착수점검과 연결됩니다.

프로젝트 유형 · 특성 선택 — Activity별 진행 방식과 정리 결과가 자동으로 조정됩니다

Sprint 0 사전 정렬

7개 Activity를 시작하기 전에 확인할 기본 정보

기존 7개 Activity를 대체하거나 새로운 Activity를 추가하는 것이 아니라, A1~A7을 시작하기 전에 PM이 확인해 두면 좋은 계약·제안 정보, 프로젝트 배경, 고객 기대효과 가설, 범위, 수행계획, 초기 미정 항목을 정리하는 준비 단계입니다. 결과물을 완성하는 것이 아니라, Sprint 0에서 무엇을 확인·결정해야 하는지, 본수행 착수 전 무엇이 미확정인지 정리하는 데 목적이 있습니다.

1계약·제안 정보 수집

RFP, 제안서, 계약서/SOW, 과업범위, 기술협상 내역, 사전 미팅 기록을 수집합니다.

2프로젝트 기본 맥락 정리

사업명, 사업기간, 추진 배경, 고객 기대사항, 성공 기준 초안을 정리합니다.

3프로젝트 유형 및 특성 확인

PoC / 본사업 여부와 일반AX / 공공AX / 금융AX 특성을 확인하여 Sprint 0 점검 깊이를 결정합니다.

4범위(Scope In/Out) 초안 정리

구축 대상 업무, 제외 범위, 주요 가정, 제약사항을 정리합니다.

5Sprint 0 수행계획 수립

Sprint 0 기간, 참석자, 수행할 Activity, 검토 일정, 책임자를 정의합니다.

6초기 미정 항목 등록

미확정 범위, 고객 의사결정, 환경·권한, 데이터, 보안, 일정 리스크를 등록합니다.

7본수행 착수 판단 기준 초안 정의

Sprint 0 종료 시 착수 가능 / 조건부 착수 / 착수 보류를 판단할 기준 초안을 수립합니다.

Seven Sub Activities

7개 하위 Activity

Sprint 0에서는 본수행 착수 전 반드시 확인해야 할 수행방식, 책임구조, 운영 리듬, 산출물 기준, 환경·권한, 고객 준비도, 리스크, 착수 가능 여부를 정리합니다. 각 카드를 클릭하면 상세 Play 페이지로 이동합니다.

A1
수행방식 및 Sprint 운영 기준 정렬
AXD, Agile, Hybrid 중 본 프로젝트에 맞는 수행방식을 정하고 Sprint 운영 기준을 합의합니다.
이 프로젝트는 어떤 방식으로 수행할 것인가?
A2
참여자 역할과 의사결정 구조 확인
고객, 수행사, 협력사, 지원조직의 참여자와 의사결정 구조를 확인하고 책임 공백을 점검합니다.
누가 무엇을 결정하고 책임지는가?
A3
커뮤니케이션 및 운영 리듬 수립
보고, 회의, Sprint Review, 의사결정, Action Item 관리 방식을 정합니다.
프로젝트 운영 리듬은 어떻게 가져갈 것인가?
A4
산출물·검토·승인 기준 정렬
프로젝트 산출물의 작성, 검토, 승인, 제출 기준을 정리합니다.
어떤 산출물을 언제, 누가 검토·승인하는가?
A5
환경·도구·접근권한 준비상태 확인
본수행에 필요한 협업도구, 저장소, 개발·검증 환경, 고객 시스템 접근권한을 확인합니다.
Sprint 1을 시작할 최소 환경과 권한이 준비되었는가?
A6
고객 준비도·보안·리스크 점검
고객 준비상태, 데이터, 보안, 의사결정, 외부 의존사항을 확인하고 본수행 지연 요인을 미정 항목으로 관리합니다.
본수행을 지연시킬 고객·보안·데이터 리스크는 무엇인가?
A7
본수행 착수점검 및 미결사항 이관
Sprint 0 결과를 종합해 본수행 착수 가능 여부를 판단하고 미결사항을 후속 단계로 이관합니다.
지금 본수행에 착수해도 되는가?
고객 업무 이해 관점

고객이 실제로 원하는 효과를 확인하기 위한 활동

고객 요구사항을 그대로 개발로 연결하지 않고, 고객이 실제로 불편해하는 업무와 기대하는 효과를 먼저 확인합니다. 별도 Activity를 추가하지 않고, 기존 7개 Activity 중 필요한 지점에서 함께 확인합니다.

Activity확인할 내용
A2고객 업무를 실제로 이해하려면 누구와 확인해야 하는지 대상자를 정리합니다
A3인터뷰나 업무 확인을 언제 진행할지 일정에 반영합니다
A5업무 화면, 자료, 샘플 데이터를 볼 수 있는지 접근 가능성을 확인합니다
A6고객 참여, 데이터 제공, 보안 등 확인을 막을 수 있는 리스크를 점검합니다
A7Sprint 1에서 가장 작게 먼저 확인해 볼 업무 시나리오를 정리합니다
참고: 각 Activity 상세 화면의 "고객 가치 발견(Value Discovery) 관점" 섹션에서 더 자세한 확인 기준을 볼 수 있습니다.

운영 원칙: Sprint 0에서 모든 내용을 완성하려고 하지 않습니다. 사전 자료를 기반으로 60~70% 초안을 준비하고, Sprint 0에서는 검토·합의·결정·Owner 지정에 집중합니다. 미완료 항목은 이슈·리스크·의사결정 항목과 본수행 착수점검표로 연결합니다.
