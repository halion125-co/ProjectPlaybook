Sprint 0 Overview

Sprint 0 Activity Playbook

Sprint 0 본문과 7개 하위 Activity별로 Task, Play 방식, 담당자, 수집정보, 의사결정, 산출물 템플릿을 정의한 HTML Pack입니다.

Agile 용어 유지: Sprint 0 · Backlog · DoR · DoD · Acceptance Criteria AXD 수행계획서 Chapter 연결 본수행 착수점검 착수 판단 입력
Playbook Output
Task를 수행하면 템플릿이 생성되고, 템플릿 상태가 본수행 착수 판단으로 연결됩니다.

이 HTML Pack은 전통적 사업수행계획서 산출물을 대체하기보다, Sprint 0에서 Hybrid 수행에 필요한 입력값과 실행 산출물을 구조화하는 용도입니다.

Sprint 0 Core

Sprint 0 착수 준비 및 사업 맥락 정렬

Sprint 0 본문은 7개 하위 Activity가 시작되기 전에 프로젝트 배경, 목표, 범위, Sprint 0 운영 계획을 정렬하는 Play입니다.

1계약·제안 정보 수집

RFP, 제안서, 계약서/SOW, 기술협상 내역, 사전 미팅 기록을 수집합니다.

2Project Brief 작성

사업명, 사업기간, 추진 배경, 고객 기대성과, 성공 기준을 정리합니다.

3Scope 기준선 작성

Scope In/Out, 구축 대상 업무, 개발 대상 업무, 주요 가정·제약사항을 정리합니다.

4Sprint 0 수행계획 수립

Sprint 0 기간, 참석자, 세션, 산출물, 책임자를 정의합니다.

5Open Item 초기 등록

미확정 범위, 의사결정, 환경·권한, 고객 준비 이슈를 초기 Open Item으로 등록합니다.

Execution Flow

운영 흐름

Activity를 수행하면서 산출물이 생성되고, 상태값이 본수행 착수 판단으로 이어집니다.

1

사전정보 수집

2

Activity별 Play 수행

3

템플릿 생성·보완

4

본수행 착수점검 착수 판단

5

사업수행계획서 반영

운영 원칙: Sprint 0에서 모든 내용을 완성하려고 하지 않습니다. 사전 자료를 기반으로 60~70% 초안을 준비하고, Sprint 0에서는 검토·합의·결정·Owner 지정에 집중합니다. 미완료 항목은 RAID와 본수행 착수점검표로 연결합니다.
적용 범위 · Sprint 0와 7개 하위 Activity 상세 Play 정의. AXD 사업수행계획서 Chapter와 연결되며, 본수행 착수점검 착수 점검에 사용됩니다.
