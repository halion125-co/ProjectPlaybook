A7본수행 착수점검 및 미결사항 이관

Sprint 0에서 생성된 산출물을 종합하여 본수행 착수 가능 여부를 판단하고, 미결사항을 Sprint 1 또는 사업수행계획서 보완 항목으로 이관합니다.

공통 운영 기준: Sprint 0 · 작업 목록 · 작업 시작·완료 기준 · 검수·수용 기준 AXD 수행계획서 Chapter 연결 본수행 착수점검 착수 판단 입력
Playbook Output
Task를 수행하면 템플릿이 생성되고, 템플릿 상태가 본수행 착수 판단으로 연결됩니다.

이 HTML Pack은 전통적 사업수행계획서 산출물을 대체하기보다, Sprint 0에서 Hybrid 수행에 필요한 입력값과 실행 산출물을 구조화하는 용도입니다.

Task

수행 Task

이 Activity를 수행할 때 실제로 처리해야 할 작업입니다.

1오픈·전환 고려사항 식별

업무전환, 데이터전환, 시스템 오픈, 안정화, 운영전환 필요 여부를 확인합니다.

2교육·기술이전 방향 확인

사용자 교육, 운영자 교육, 관리자 교육, 기술이전 대상과 시점을 확인합니다.

3유지보수·안정화 방향 확인

무상 하자보수, 안정화 지원, 운영지원 범위의 계약상 조건을 확인합니다.

4본수행 착수 준비 Review

앞선 Activity 정리 결과를 종합해 착수 가능/조건부 착수/착수 보류를 판단합니다.

5Open Item 이관

미결사항을 Sprint 1 작업 목록, 이슈·리스크 목록, 사업수행계획서 보완 항목으로 이관합니다.

Play 방식

어떻게 수행하는가

  • 본수행 착수점검표는 새로운 문서를 작성하는 회의가 아니라, 앞선 Activity 결과를 검증하는 절차입니다.
  • 모든 항목이 완료되어야 착수 가능이 되는 것은 아니며, 조건부 착수 기준을 명확히 합니다.
  • 착수 보류 항목은 고객 PM과 Sponsor가 볼 수 있도록 별도 요약합니다.
  • 사업수행계획서에 반영할 항목과 Sprint 1에서 계속 추적할 항목을 분리합니다.
